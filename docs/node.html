Doesn't Node already have a module loader? § 1

Yes Node does. That loader uses the CommonJS module format. The CommonJS module format is non-optimal for the browser, and I do not prefer some of the trade-offs made in the CommonJS module format. By using RequireJS on the server, you can use one format for all your modules, whether they are running server side or in the browser. That way you can preserve the speed benefits and easy debugging you get with RequireJS in the browser, and not have to worry about extra translation costs for moving between two formats.

Can I use server modules already written in the CommonJS module format? § 2

Yes! The Node adapter for RequireJS, called r.js, automatically converts CommonJS modules on the fly to a format usable by RequireJS, as they are loaded, so you can use your existing Node-based modules without having to hand-modify them.

However, the top most app file, the one you pass to Node on startup, needs to be coded to the RequireJS API. That file is evaluated as-is, no automatic format conversion.

You can use modules installed by npm, or use the Configuration Options for RequireJS in your top level app file to configure paths, package pages, and even a different baseUrl for your modules.

RequireJS will use its configuration first to find modules, then use require.paths to search for the module, then if it is not found, it will look in the internal cache of modules built into Node.

RequireJS does not support require.main and the module free variable only contains an id property. It does not define a require.async method, since require() in RequireJS can accept equivalent arguments.

Even though RequireJS is an asynchronous loader in the browser, the RequireJS Node adapter loads modules synchronously in the Node environment to match the default loading behavior in Node.

Finally, RequireJS in Node can only load modules that are on the local disk -- fetching modules across http, for instance, is not supported at this time.

How do I use it? § 3

Download r.js from the the download page and place it on your disk somewhere. Then run this command:

node path/to/r.js myNodeApp.js

This assumes you are in the directory that contains myNodeApp.js, your top-level Node application file.

That is it!

If you want to try a sample app that works with r.js, see the index.js Hello World app on the download page.

If you get a load failure, you can pass debug after r.js to get some information on what modules were attempted to be loaded and from what location:

node path/to/r.js debug myNodeApp.js

If you find you have a problem, and want to report it, use the RequireJS GitHub Issues page. If you can attach the output of the debug run, that would be helpful.

If you want to discuss the RequireJS-Node integration, you can use the RequireJS Group.
